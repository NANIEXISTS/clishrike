# Shrike CLI — Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 1.0.0-poc (Beta) | ✅ Active |

## Reporting a Vulnerability

If you discover a security vulnerability in Shrike CLI (including the website, license server, or distribution packages), **please do not open a public GitHub issue.**

Report it directly and privately to:

**support@shrike.pro**

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact

We will acknowledge your report within **48 hours** and provide a fix timeline within **7 days** for critical issues.

## Scope

This policy covers:
- The Shrike CLI binary and source package
- The `cli.shrike.pro` web property
- The license activation API (`api.shrike.pro`)

This policy does **not** cover:
- Third-party dependencies (report these upstream)
- Issues in the Stripe API itself
